Why den

Guardrails on the box,
not on the agent

Lock down a disposable microVM once — its filesystem, its network — then let any agent run unattended inside it. The blast radius stops at the sandbox.

Full YOLO, safely

Flip on bypass-permissions and stop approving every command. The agent can install anything, run anything, touch any file — it's sealed inside a microVM it can't escape, so a bad call never reaches your machine.

Every agent, first-class

Full agent TUIs over a real PTY — bring the one you already use.

Isolated by default

Each sandbox is its own microVM with a separate filesystem and network. Your host stays untouched.

Network policies

Decide exactly which domains an agent can reach. Everything else is blocked — no exfiltration, no surprises.

Ephemeral, and yours to shape

Spin up a disposable sandbox in seconds, then layer in your own tools, credentials and policies with kits and mixins. Throw it away when you're done.

Watch them all work

Every sandbox, its agent, status and uptime in one dashboard — run a whole fleet at once.

Get started

Up and running in minutes

den sits on top of your existing sbx setup. If you've used Docker Sandboxes, you're already halfway there.

1

Install den

Download the app, drop it in Applications. No Homebrew, no npm install, no setup wizard.

2

Connect to sbx

Sign in with your Docker account. den picks up your sbx credentials automatically.

3

Create a sandbox

Pick a base image, tweak resources if you like, and launch. Your isolated microVM is ready in seconds.

4

Let it run

Launch Claude, Codex, Antigravity or OpenCode, turn off the prompts, and let it build, explore and ship — unattended.
