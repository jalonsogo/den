What it does

A real window into sbx

Launch sandboxes, drive agents, browse files, and manage network policy — without living in the terminal.

Sandbox dashboard

Every sandbox with its agent, status and uptime in one window — plus a menu-bar menu for quick access.

Agent & shell terminals

Full agent TUIs — Claude Code, Codex, Cursor, Gemini — and an interactive shell, over a real PTY.

Files, inline

Browse the sandbox workspace, open files in an editor, and see the agent's changes with live git status.

Network policy

See exactly which domains a sandbox can reach, and add allow rules — without leaving the app.

Projects & kits

Group sandboxes by workspace and layer tools, credentials and config with sbx kits and mixins.

Isolated by default

Each sandbox is its own microVM with a separate filesystem and network. Your host stays untouched.

Get started

Up and running in minutes

den sits on top of your existing sbx setup. If you've used Docker Sandboxes, you're already halfway there.

1

Install den

Download the app, drop it in Applications. No Homebrew, no npm install, no setup wizard.

2

Connect to sbx

Sign in with your Docker account. den picks up your sbx credentials automatically.

3

Create a sandbox

Pick a base image, tweak resources if you like, and launch. Your isolated microVM is ready in seconds.

4

Run your agents

Launch Claude or any AI agent into the sandbox. Watch it build, explore, and ship from the dashboard.
